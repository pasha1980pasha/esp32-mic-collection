Набор программ для ESP32, которые позволяют подключать аналоговый микрофон HW-484 и работать со звуком. Скетчи покрывают всё: от простого теста микрофона до распознавания голосовых команд и стриминга звука в браузер.

Какие есть версии
1. Простой тест микрофона
Самый простой скетч. Просто показывает значение с микрофона на веб-странице в виде большого числа и цветной полоски. Подходит для быстрой проверки, работает ли микрофон и правильно ли он настроен.

2. Аналоговый микрофон с VU-метром
Более продвинутая версия. Показывает не просто текущее значение, а реальную громкость звука. Есть VU-метр (полоска, которая прыгает в такт звуку), кнопки включения/выключения и калибровки. Удобно для визуального контроля уровня сигнала.

3. Аудио стрим через WAV
Интересная штука. Создаёт аудиопоток, который можно слушать прямо в браузере. На странице есть кнопка "СЛУШАТЬ" — нажимаешь, и в динамиках начинает играть звук с микрофона в реальном времени. Работает на любой ОС, без дополнительных плагинов.

4. WebSocket аудио стрим
То же, что и предыдущее, но через WebSocket. Позволяет передавать звук в реальном времени с минимальной задержкой. Использует Web Audio API для воспроизведения. Подходит для приложений, где важна низкая задержка.

5. Осциллограф звука
Визуализирует звуковую волну. На веб-странице отображается график сигнала с микрофона — настоящий осциллограф, который показывает форму звуковой волны. Есть цветовая индикация уровня громкости.

6. Распознавание голосовых команд
Самая продвинутая версия. Анализирует записанный звук и распознаёт четыре команды: «СТОП», «ВКЛЮЧИ», «ВЫКЛЮЧИ», «ПРИВЕТ». Работает на основе анализа громкости и частоты — не требует нейросетей и сложных библиотек. Показывает уверенность распознавания в процентах. Есть визуализатор звука и автоматическое начало записи при обнаружении голоса.

Как это подключить
Микрофон подключается к ESP32 всего тремя проводами:

OUT → GPIO14 (это пин, с которого читается сигнал)

VCC → 3.3V (питание)

GND → GND (земля)

Перед использованием нужно настроить потенциометр на модуле HW-484. В тишине значение АЦП должно быть около 2048, при разговоре — прыгать в диапазоне 1500–2500.

Как запустить
Любой скетч загружается через Arduino IDE. После загрузки ESP32 создаёт Wi-Fi точку доступа. В зависимости от версии, названия и пароли могут отличаться:

ESP32-Voice, пароль 12345678

ESP32-Mic-Test, пароль 12345678

ESP32-MIC, пароль 12345678

ESP32-TEST, пароль 12345678

Подключаешься к этой сети телефоном или компьютером, открываешь браузер, вводишь IP адрес (обычно 10.0.0.1 или 192.168.4.1), и открывается веб-интерфейс. Дальше всё просто — нажимаешь кнопки, говоришь команды, смотришь графики.

Зачем это нужно
Эти скетчи можно использовать как основу для голосового управления устройствами, для систем безопасности (детектор звука), для визуализации музыки, для тестирования микрофонов, для учебных проектов по обработке звука на микроконтроллерах. Всё работает локально, без интернета, только через Wi-Fi ESP32.

Что важно знать
Все скетчи написаны на Arduino (C++), используют стандартные библиотеки, не требуют внешних сервисов. Работают на любом ESP32, не только на ESP32-S3. Микрофон HW-484 стоит копейки и легко подключается. Если нужно распознавание точнее — можно доработать пороги в коде, они вынесены в отдельные переменные.

В общем, если есть ESP32 и микрофон — можно за пару минут сделать полезное устройство, которое слышит и понимает голос.
